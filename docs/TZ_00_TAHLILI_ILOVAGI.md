## 1. Чӣ чизҳои МУҲИМ дар таҳлили аввал НАБУДАНД?

Таҳлили аввал асосро (анбор + касса + приход) дуруст гирифт, вале барои
дорухонаи воқеӣ инҳо ҳам **ҳатмӣ** ё **хеле муҳим**-анд:

### 1.1. Модулҳои набудаи МУҲИМ

| # | Модул | Чаро лозим аст? | Дараҷа |
|---|-------|-----------------|--------|
| 1 | **Кассаи смена (Кассовая смена)** | Кушодан/бастани смена, X- ва Z-ҳисобот, инкассатсия. Бе ин назорати нақд нест. | 🔴 Ҳатмӣ |
| 2 | **Бозгашти мол аз мизоҷ (Возврат)** | Мизоҷ доруро бармегардонад → бояд пул баргардад ва бақия зиёд шавад. | 🔴 Ҳатмӣ |
| 3 | **Списание (Write-off)** | Доруи мӯҳлаташ гузашта, шикаста, вайрон — бояд аз анбор тоза шавад бо сабаб. | 🔴 Ҳатмӣ |
| 4 | **Инвентаризатсия** | Ҳисобкунии воқеии анбор ва мутобиқкунӣ бо база (барзиёд/камӣ). | 🔴 Ҳатмӣ |
| 5 | **FEFO (First Expired First Out)** | Ҳангоми фурӯш аввал доруи мӯҳлаташ наздик мефурӯшад. Барои дору — қонун. | 🔴 Ҳатмӣ |
| 6 | **Бозгашт ба таъминкунанда** | Доруи нодуруст/мӯҳлаташ наздикро ба таъминкунанда баргардонидан. | 🟡 Муҳим |
| 7 | **Перемещение байни филиалҳо (Transfer)** | Дору аз як филиал ба дигараш. Барои шабака ҳатмӣ. | 🟡 Муҳим |
| 8 | **Минималии бақия + автозаказ** | Огоҳӣ вақте дору кам мешавад; пешниҳоди фармоиш ба таъминкунанда. | 🟡 Муҳим |
| 9 | **Идоракунии нарх (Pricing/Markup)** | Қоидаҳои наценка, тағйири нарх, таърихи нарх. | 🟡 Муҳим |
| 10 | **Тахфиф ва аксияҳо (Discounts)** | Тахфиф ба чек, ба мол, карти муштарӣ. | 🟢 Дилхоҳ |
| 11 | **Барномаи вафодорӣ (Loyalty)** | Картаи муштарӣ, бонусҳо. | 🟢 Дилхоҳ |
| 12 | **Аналогҳои дору** | Нишон додани доруҳои ивазшаванда ҳангоми набудан. | 🟢 Дилхоҳ |
| 13 | **Журнали амалиёт (Audit log)** | Кӣ, кай, чиро тағйир дод — барои назорат ва бехатарӣ. | 🔴 Ҳатмӣ |
| 14 | **Фискалӣ / маркировка (Data Matrix)** | Агар дар Тоҷикистон ҳатмӣ бошад — ба сканер ва қонун таъсир мекунад. | ⚠️ Тафтиш кардан |

### 1.2. Қоидаҳои тиҷоратии хоси дорухона (Business Rules)

Инҳо қоидаҳое, ки система БОЯД риоя кунад:

1. **Учёт партиявӣ (батч):** ҳар дору на танҳо «миқдор», балки «миқдор дар ҳар
   партия бо мӯҳлати худаш» дорад. Бе ин назорати мӯҳлат имконнопазир аст.
2. **FEFO ҳангоми фурӯш:** система автоматӣ аз партияи мӯҳлаташ наздиктар кам мекунад.
3. **Манъи фурӯши мӯҳлаташ гузашта:** доруи мӯҳлаташ гузаштаро система намефурӯшад
   (танҳо ба списание мегузарад).
4. **Огоҳӣ дар бораи мӯҳлат:** N рӯз пеш аз мӯҳлат (мас. 90/30 рӯз) огоҳӣ.
5. **Доруи ретсептӣ:** ҳангоми фурӯши доруи ретсептӣ — қайди ретсепт (ё огоҳӣ).
6. **Бақия ҳеҷ гоҳ манфӣ намешавад:** фурӯш аз бақияи мавҷуда зиёд иҷозат нест.
7. **Нарх дар лаҳзаи фурӯш қайд мешавад:** агар нарх баъд тағйир ёбад, чеки кӯҳна тағйир намеёбад.
8. **Ҳар амалиёт ба смена ё ҳуҷҷат баста аст:** фурӯш танҳо ҳангоми смена кушода.

## 2. Талаботи функсионалӣ (пурра, бо модулҳо)

### MODUL 1 — Аутентификатсия ва корбарон
- Воридшавӣ бо логин/парол, JWT токен.
- Нақшҳо: **Admin, Менеджер, Фурӯшанда (касса), Анбордор**.
- Идоракунии корбарон (сохтан, ғайрифаъол кардан, ивази парол).
- Ҳар корбар ба як ё якчанд филиал баста.

### MODUL 2 — Справочникҳо (Маълумотномаҳо)
- Доруҳо (номенклатура/товар) — корти дору.
- Справочники МНН (база-эталони доруҳо).
- Гурӯҳҳои дору, истеҳсолкунандагон, таъминкунандагон.
- Воҳидҳои ченак, филиалҳо.
- Аналогҳо (дилхоҳ).

### MODUL 3 — Қабули мол (Приход)
- Сохтани накладная: таъминкунанда, сана, рақам.
- Сатрҳо: дору, миқдор, **серия**, **мӯҳлат**, нархи харид, нархи фурӯш, НДС.
- Скани штрих-код ҳангоми қабул.
- Тасдиқ (проведение) → бақия ва партия автоматӣ зиёд мешавад.
- Чопи этикетка/ценник аз приход.

### MODUL 4 — Анбор (Склад)
- Бақия аз рӯи филиал ва партия.
- Ҷустуҷӯ бо ном/штрих-код/МНН.
- Мӯҳлатҳои наздик — рӯйхати огоҳӣ.
- Таърихи ҳаракати ҳар дору (приход/расход/возврат/списание).

### MODUL 5 — Касса (POS / Фурӯш)
- Кушодани смена → фурӯш → бастани смена (Z-отчёт).
- Илова кардани дору ба чек (скан ё ҷустуҷӯ).
- FEFO автоматӣ.
- Тахфиф, тарзи пардохт (нақд/корт), ҳисоби қарзи харидор.
- Чопи чек (ESC/POS).
- Бозгашти фурӯш (возврат).

### MODUL 6 — Амалиёти анборӣ
- Списание (мӯҳлат гузашта/вайрон) бо сабаб.
- Инвентаризатсия (санҷиш + мутобиқкунӣ).
- Перемещение байни филиалҳо.
- Бозгашт ба таъминкунанда.

### MODUL 7 — Ҳисоботҳо
- Фурӯш (рӯз/ҳафта/моҳ, аз рӯи филиал, фурӯшанда, дору).
- Бақияи анбор ва арзиши он.
- Доруҳои мӯҳлаташ наздик/гузашта.
- Доруҳои камшуда (зери минимум).
- Фоида (нархи фурӯш − нархи харид).
- Z-ҳисоботи смена, ABC-таҳлил (дилхоҳ).

### MODUL 8 — Шабака (бисёрфилиалӣ) — ⏸️ ФАЗАИ ОХИР (1 филиал дар оғоз)
> Дар оғоз 1 филиал аст, бинобар ин ин модул ба охир мегузарад. Вале база аз
> аввал `BranchId` нигоҳ медорад, то баъдтар бе тағйири архитектура васеъ шавад.
- Анбори марказӣ ва тақсимот.
- Ҳисоботи умумии шабака.
- Нархи ягона ё нархи ҳар филиал.
- Перемещение байни филиалҳо (transfer).

### MODUL 9 — Системавӣ
- Журнали амалиёт (audit log).
- Backup автоматӣ.
- Танзимот (огоҳӣ, мӯҳлат, наценка).

## 3. Талаботи ғайрифунксионалӣ (Non-functional)

| Талабот | Тавсиф |
|---------|--------|
| **Кор бе интернет (offline)** | Касса бояд ҳангоми қатъи интернет кор кунад: фурӯш дар кэши маҳаллӣ нигоҳ дошта, баъд синхрон шавад. |

| **Бехатарӣ** | JWT, парол hash (bcrypt/Identity), HTTPS, ҳуқуқ аз рӯи нақш. |
| **Эътимоднокӣ** | Backup ҳаррӯза, транзаксияҳои ACID (бақия ҳеҷ гоҳ хато нашавад). |
| **Васеъшавандагӣ** | Аз 1 филиал то 50+ филиал бе тағйири архитектура. |
| **Журнал/Лог** | Хатоҳо ва амалиёти муҳим сабт шаванд (Serilog). |
| **Локализатсия** | Тоҷикӣ/Русӣ (UI), форматҳои сана/пул. |
| **Қобили нигаҳдорӣ** | Clean Architecture, тестҳо, ҳуҷҷати API (Swagger). |

      мешаванд; `pubspec.yaml` → `platforms: { windows:, macos: }`.

      дашборд, ҷадвалҳои зич вале равшан, огоҳии барҷастаи мӯҳлат/камшуда, checkout-и зуди

> кадом технология ва дастгоҳ лозим аст, дар чанд муддат тайёр мешавад.

## 1. Чӣ хел кор мекунем? (Методология / Approach)

Кор ба **марҳилаҳо (этапҳо)** тақсим мешавад — аввал асос, баъд модулҳо як ба як.
Ин роҳ хатарро кам мекунад ва имкон медиҳад, ки ҳар марҳиларо санҷем.

**Принсипҳои асосӣ:**
- **MVP аввал** — версияи минималии корӣ (анбор + касса), баъд иловаҳо.
- **Як филиал аввал** — аввал як дорухонаро пурра кор кунонем, баъд ба шабака васеъ кунем.
- **API-first** — аввал сервер (backend) ва API-ро тайёр мекунем, баъд Flutter ба он пайваст мешавад.
- **Тест дар ҳар марҳила** — ҳар модул пеш аз гузаштан санҷида мешавад.

**Тартиби корҳо:**
1. Таҳлил ва тарҳрезии маълумотгоҳ (database design)
2. Сервери .NET — асос: аутентификатсия, API скелет, пайвасти база
3. Маълумотномаҳо (справочники): доруҳо, гурӯҳҳо, истеҳсолкунандагон, таъминкунандагон
4. Модули қабули мол (приход) — роҳи асосии ворид кардани доруҳо
5. Модули анбор (склад): бақия, мӯҳлат, огоҳиҳо
6. Модули касса (POS): фурӯш, скан, чек
7. Ҳисоботҳо
8. Корбарон ва ҳуқуқҳо (роли admin/seller/storekeeper)
9. Функсияҳои шабака (бисёрфилиалӣ): анбори марказӣ, ҳисоботи умумӣ
10. Тест, ислоҳ, ҷорӣ кардан (deployment)

## 2. Доруҳоро чӣ хел ба система ворид мекунем? (Data entry)

Ин яке аз муҳимтарин саволҳост. Панҷ роҳ ҳаст — мо ҳамаашро дастгирӣ мекунем:

### Роҳи 1 — Дастӣ (Manual) ✍️
Корбар номи дору, нарх, мӯҳлатро дастӣ ворид мекунад.
- ✅ Содда, барои оғоз ва доруҳои нав.
- ❌ Суст барои миқдори зиёд.

### Роҳи 2 — Қабули мол / Приходная накладная 📦 (асосӣ)
Вақте дору аз таъминкунанда меояд, ҳамроҳаш **накладная** (рӯйхати мол) меояд.
Корбар онро ба система ворид мекунад — ҳар сатр як дору бо: миқдор, серия,
мӯҳлат, нархи харид ва нархи фурӯш.
- ✅ Ин роҳи табиӣ ва асосии воридкунӣ дар дорухона аст.
- ✅ Бақияи анбор худкор зиёд мешавад.

### Роҳи 3 — Скани штрих-код 📷
Ҳангоми қабул штрих-коди дору скан карда мешавад:
- Агар дору дар база бошад → худкор пайдо мешавад.
- Агар набошад → корти нав сохта, штрих-код ба он баста мешавад.
- ✅ Зуд ва бехато.

### Роҳи 4 — Импорт аз Excel / CSV 📊
Рӯйхати калони доруҳоро якбора аз файли Excel ворид кардан.
- ✅ Барои кӯчонидани база аз системаи кӯҳна ё рӯйхати таъминкунанда.

### Роҳи 5 — Базаи тайёри доруҳо (Drug reference) 📚
Як **справочники доруҳо** (номгӯи стандартии доруҳо бо МНН, истеҳсолкунанда)
тайёр карда мешавад, то корбар аз нав ҳама чизро нанависад — танҳо аз рӯйхат
интихоб кунад. Ин база бо мурури кор бой мешавад.

> **Тавсия:** Роҳи асосӣ = **Приход (накладная) + скани штрих-код**. Дастӣ ва Excel
> ҳамчун ёрирасон. Справочник аз рӯзи аввал ба таври худкор аз приходҳо пур мешавад.

**Маълумоти ҳатмӣ барои ҳар як дору:**

| Ном | Парацетамол 500мг №10 |

## 3. Кадом технологияҳо? (Tech Stack)

### Backend — .NET
| Қисм | Технология |
|------|-----------|
| Framework | **ASP.NET Core Web API** (.NET 8/9 LTS) |
| Забон | C# |
| ORM (кор бо база) | **Entity Framework Core** |
| Маълумотгоҳ | **PostgreSQL** (ройгон, пурқувват) ё SQL Server |
| Аутентификатсия | **JWT** + ASP.NET Core Identity |
| Ҳуҷҷати API | **Swagger / OpenAPI** |
| Real-time (ихтиёрӣ) | **SignalR** — навсозии зинда байни филиалҳо |
| Архитектура | Clean Architecture (API / Application / Domain / Infrastructure) |

### Frontend — Flutter
| Қисм | Технология |
|------|-----------|
| Framework | **Flutter** (Windows desktop аввал) |
| Забон | Dart |
| State management | **Riverpod** ё **Bloc** |
| Кор бо API | **Dio** (HTTP client) |
| Кэши маҳаллӣ / offline | **Drift** (SQLite) ё **Hive** |
| Навигатсия | **go_router** |
| Скани штрих-код | USB scanner = вуруди клавиатура; камера = `mobile_scanner` |
| Чоп кардани чек | **esc_pos_utils** (принтери термалӣ) ё `pdf` + `printing` |
| Ҷадвал/UI | Material 3, `data_table_2` барои рӯйхатҳои калон |

### Инфраструктура
| Қисм | Интихоб |
|------|---------|
| Ҷойгиркунии сервер | Сервери офисӣ (on-premise) ё VPS облачӣ |
| Системаи сервер | Windows Server ё Linux (Ubuntu) |
| Backup | Худкор — ҳаррӯза, нусхаи база |
| Идоракунии код | **Git** (GitHub / GitLab) |
| Контейнер (ихтиёрӣ) | **Docker** — барои осон ҷойгир кардани сервер |

## 4. Кадом дастгоҳҳо лозим аст? (Hardware)

### Барои сервери марказӣ (як дона)
- **Сервер** ё VPS: CPU 4+ ядро, RAM 8–16 ГБ, SSD 256+ ГБ.
- **Интернети устувор** бо IP-и доимӣ (статикӣ) — барои пайвасти филиалҳо.
- **UPS** (манбаи барқи захиравӣ) — барои нагузоштани қатъи база.

### Барои ҳар як филиал (дорухона)
| Дастгоҳ | Тавсиф | Ҳатмӣ? |
|---------|--------|--------|
| 💻 **Компютер/ноутбук** | Windows 10/11, барои кассаи Flutter | Ҳа |
| 📷 **Сканери штрих-код** | USB, 1D (ё 2D барруи QR/DataMatrix) | Ҳа |
| 🧾 **Принтери чек** | Термалӣ 58мм ё 80мм, ESC/POS | Ҳа |
| 🌐 **Интернет** | Устувор, барои пайваст ба сервер | Ҳа |
| 🔌 **UPS** | Барқи захиравӣ | Тавсия |
| 🏷️ **Принтери этикетка** | Чопи штрих-код барои молҳои бе код | Ихтиёрӣ |
| 💵 **Ящики касса** (cash drawer) | Барои пул | Ихтиёрӣ |
| 🖥️ **Дисплейи мизоҷ** | Нархро ба мизоҷ нишон медиҳад | Ихтиёрӣ |

> **Эзоҳ:** Сканери штрих-коди USB ҳамчун клавиатура кор мекунад — барномасозии
> махсус талаб намекунад. Принтери чек ESC/POS-ро дастгирӣ кунад, кофист.

## 6. Хулоса (Conclusion)

1. **Технология аниқ шуд:** Backend = .NET (ASP.NET Core + EF Core + PostgreSQL),
   Frontend = Flutter (Windows desktop), пайваст тавассути REST API + JWT.
2. **Воридкунии доруҳо:** роҳи асосӣ — қабули мол (приход) + скани штрих-код;
   ёрирасон — дастӣ, Excel-импорт ва справочники худпуршаванда.
3. **Дастгоҳҳо:** сервери марказӣ + дар ҳар филиал компютер, сканер, принтери чек,
   интернет (UPS тавсия).
4. **Муддат:** MVP ~2–2.5 моҳ, версияи пурра ~4–5 моҳ (як дастур).
5. **Қадами оянда:** тасдиқи ин нақша → тарҳрезии муфассали маълумотгоҳ (ERD) →
   оғози навиштани сервери .NET.
